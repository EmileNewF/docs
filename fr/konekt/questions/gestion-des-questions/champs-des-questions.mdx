Une bonne partie de la puissance de Newforma Konekt est basée sur une bonne assignation des attributs de question; la plupart peuvent être personnalisés par projet dans les paramètres du projet.

La chose la plus importante à garder à l'esprit lorsque vous définissez et attribuez ces paramètres est que vous allez filtrer les questions pour les rapports de coordination, vérifiez que votre projet est sur une bonne voie et plus. Penser à l'avance à ce dont vous aurez besoin pour effectuer une bonne trie des questions et afin de définir les bons paramètres.

Connexe: Découvrez comment nous avons configuré les attributs de notre projet à partir de l'article [***<u>Paramètres du projet</u>***](https://konekt.help.newforma.com/4408486565645-hubs-projects/360008384612-projects/360041898991-project-settings/).

### **Titre**

Le titre d'une question vous permet de la décrire. Ce champ est obligatoire.

### **Description**

La description d'une question vous permet de donner des instructions ou d'ajouter d'autres informations complémentaires.

### **Statut**

L'attribut statut informe les utilisateurs de l'état de la question.

### **Type**

Le champ Type sert à fournir des informations supplémentaires sur la nature d'une question. Les questions publiées dans Newforma Konekt ne sont pas nécessairement seulement que des interférences (Conflits/Collisions/Clashes). Il peut s'agir de RFI, de commentaires, de demandes, de conflits de cours, de questions, de validation de données, de demandes de remplacement, de défauts, de problèmes financiers, d'erreurs ou d'omissions. Certaines valeurs par défaut sont déjà attribuées lors de la création de projets et celles-ci peuvent être modifiées ou ajoutées dans les paramètres du projet. Pourquoi est-ce utile? Encore une fois, vous pouvez trier vos rapports et métriques par type, ce qui vous permet de vous concentrer sur les questions à résoudre.

### **Priorité**

La priorité, qui peut aussi être renommée Sévérité ou Importance, est définie en fonction de l'impact sur le projet. N'hésitez pas à choisir, à remplir vos propres délais de résolution et à le distribuer à l'interne (ou dans votre BEP / BxP) afin que toutes les équipes soient sur la même page.

### **Discipline**

Le champ Discipline, qui peut aussi être renommé Métier, Lot de construction ou Lot technique, représente généralement des disciplines de base telles que l’architecture, la structure, le MEP ou les sous-disciplines tels que protection incendie, drainage, CVC, plomberie, structure métallique, structure en béton, etc. Gardez à l'esprit qu'il n'y a pas de lien entre ce champ et les attributs de confidentialités. Il est principalement utilisé pour les métriques, c’est-à-dire la résolution des questions par équipes de suivi. C'est également le seul attribut permettant à l'utilisateur d'appliquer plusieurs valeurs.

### **Phase**

Le champ Phase, qui peut aussi être renommé Séquence, est configuré différemment en fonction de la portée du travail ou de la phase de projet à laquelle vous êtes impliqué. Nous vous recommandons de diviser les phases par séquence de construction. Vous pouvez également les aligner par lot de travaux pour assurer un suivi spécifique avec les sous-métiers. Les phases peuvent être configurées dans les paramètres du projet.

### **Assignée à**

Le champ Assignée à est la personne qui doit résoudre la question. Dans Newforma Konekt, nous permettons aux utilisateurs d'ajouter le nom d'une personne. Cela génère un courrier électronique qui est envoyé à l'utilisateur, malgré les paramètres de notification de l'utilisateur (hebdomadaire, quotidien, mensuel). Une seule personne peut être sélectionnée pour éviter le manque de responsabilisation / manque d'action / confusion.

### **Date d'échéance**

La date d'échéance est définie par défaut a 7 jours après la création d'une question. La durée par défaut peut être modifiée dans les paramètres du projet ou affectée manuellement à une date spécifique. Il est également possible de ne pas indiquer de date d'échéance lors de la création d'une question

### **Groupe**

Le champ Groupe est un peu comme une balise (tag). Ce champ n'est pas prédéfini dans les paramètres du projet Newforma Konekt. En général, nos clients ajoutent le nom du test de collision du détective d'interférence de Navisworks ou un nom / une date de réunion de coordination pour créer des rapports pour des réunions ou des groupes de questions spécifiques.

### **Équipe impliquée**

Le champ Équipe impliquée est utilisé pour identifier la ou les équipes concernées par une question, ce qui vous permet de l'assigner à des équipes.

### **Notifier**

Le champ Notifier est l'équivalent Newforma Konekt d'envoyer un courriel en copie conforme. Les utilisateurs sélectionnés seront informés de tous les changements sur la question. Les équipes ou les personnes ayant accès à cette question peuvent être sélectionnées dans le champ notification.

### **Visibilité**

Le champ Visibilité vous permet de limiter la visibilité à l'auteur, au responsable, à l'équipe ou aux équipes dans le champ « Équipe impliquée », et aux utilisateurs/équipés notifiés. Il arrive que les questions nécessitent un degré de confidentialité plus élevé, ou que vous ne souhaitiez pas que tout le monde voit la question.

### **Attributs personnalisés**

De nombreux administrateurs de projets disposent de ce seul élément d'information qu'ils souhaitent ajouter à chaque question. Avec les attributs personnalisés, vous pouvez ajouter jusqu'à 6 attributs de question personnalisés par projet en plus des attributs prêts à l'emploi. Les champs couramment ajoutés sont les grilles et le coût.
